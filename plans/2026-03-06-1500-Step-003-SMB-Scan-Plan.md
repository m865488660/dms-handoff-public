## 1. Context（背景）

### Problem Statement
DMS currently supports only local filesystem scanning (`type=local`). Users need to scan datasets stored on network shares (SMB/CIFS), which are common in enterprise NAS environments (Synology, QNAP, Windows Server shares).

### Current Limitation
- `POST /endpoints` with `type=smb` returns HTTP 501
- Worker uses `os.walk()` which only works on locally mounted paths
- No credential storage mechanism for network authentication

### Goal
Add SMB/CIFS network scan support using a Python SMB library in the worker, without requiring OS-level mount dependencies.

## 3. Design Decisions

### 3.1 SMB Access Strategy

**Decision: Python SMB Library (`smbprotocol`) in Worker**

| Option | Pros | Cons | Decision |
|--------|------|------|----------|
| OS mount + `os.walk()` | No code change | Requires host mount, root access, fragile | Rejected |
| `smbprotocol` library | Pure Python, async-capable, maintained | New dependency | **Selected** |
| `pysmb` library | Mature, pure Python | Sync only, less maintained | Alternative |

**Implementation:**
- Use `smbprotocol` for SMB2/SMB3 support
- Create `SMBWalker` class that mirrors `os.walk()` interface
- Connection pooling per endpoint (cached during scan job)
- Graceful degradation on permission errors

### 3.2 Secret Handling Strategy

**Decision: Database JSONB + Environment Variable Fallback**

```
Storage Hierarchy:
1. Endpoint.config (JSONB in PostgreSQL)
   - host, share, roots, exclude_patterns, max_depth
2. Endpoint.credentials (NEW JSONB column)
   - username, password (stored in DB, protected by DB access control)
3. Environment Variable Override (for dev/testing)
   - SMB_{endpoint_id}_USER, SMB_{endpoint_id}_PASS
```

**Safety Measures:**
- `publish_handoff.ps1` already filters secrets via regex patterns
- Never log credentials
- API returns `has_credentials: boolean` instead of actual credentials
- Worker fetches credentials at scan time (not in task payload)

### 3.3 Integration Test Strategy

**Decision: Dockerized Samba Container**

```yaml
# docker-compose.test.yml (or extended docker-compose.yml)
smb-test:
  image: dperson/samba
  environment:
    USER: "testuser;testpass"
    SHARE: "datasets;/mount;no;no;no;testuser"
  volumes:
    - ./sample_data:/mount
  ports:
    - "445:445"
```

**Test Scenarios:**
1. Connect to SMB share with credentials
2. List directories
3. Read file metadata (size, mtime)
4. Classify datasets via signature rules
5. Handle connection failures gracefully

### 3.4 Taxonomy Assumption

**Decision: No changes to existing taxonomy**

Continue using:
- `raw` / `result` / `hybrid` (no `unknown` for valid datasets)
- `has_gcp` boolean flag
- Same signature file rules apply to SMB paths

## 4. Architecture Changes

### 4.1 Endpoint Configuration Model

**Current:**
```python
class EndpointConfig(BaseModel):
    roots: list[str]
    exclude_patterns: list[str]
    max_depth: int
```

**Proposed:**
```python
class SMBEndpointConfig(BaseModel):
    host: str                    # SMB server hostname/IP
    share: str                   # Share name
    roots: list[str]             # Root paths within share
    exclude_patterns: list[str] = ["$RECYCLE.BIN", "@eaDir", ".Trash"]
    max_depth: int = 4
    port: int = 445

class SMBCredentials(BaseModel):
    username: str
    password: str
    domain: str = ""             # Optional for AD

class EndpointConfig(BaseModel):
    # For type=local
    roots: list[str]
    exclude_patterns: list[str]
    max_depth: int

    # For type=smb (merged)
    host: Optional[str] = None
    share: Optional[str] = None
    port: int = 445
```

### 4.2 Database Schema Changes

```sql
-- Add credentials column to endpoints table
ALTER TABLE endpoints ADD COLUMN credentials JSONB DEFAULT '{}';

-- Credentials stored as: {"username": "xxx", "password": "yyy", "domain": ""}
-- Protected by DB access control; not returned by API
```

### 4.3 API Changes

| Endpoint | Change |
|----------|--------|
| `POST /endpoints` | Accept `type=smb` with SMB config fields |

| `POST /tasks/scan` | No change (endpoint_id remains payload key) |

### 4.4 Worker Changes

**New Files:**
```
worker/
├── walker/
│   ├── __init__.py
│   ├── base.py          # Abstract walker interface
│   ├── local.py         # Local filesystem walker (extracted from scan.py)
│   └── smb.py           # SMB walker implementation
├── scan.py              # Updated to use walker abstraction
└── worker.py            # No change
```

**SMBWalker Interface:**
```python
class WalkerBase(ABC):
    @abstractmethod
    def walk(self, roots: list[str], max_depth: int,
             exclude_patterns: list[str]) -> list[str]:
        """Return candidate directories."""

    @abstractmethod
    def file_exists(self, path: str) -> bool:
        """Check if file exists."""

    @abstractmethod
    def dir_exists(self, path: str) -> bool:
        """Check if directory exists."""

    @abstractmethod
    def get_file_info(self, path: str) -> FileInfo:
        """Get file size and mtime."""

class SMBWalker(WalkerBase):
    def __init__(self, host: str, share: str, credentials: SMBCredentials, port: int = 445):
        self.host = host
        self.share = share
        self.credentials = credentials
        self.port = port
        self._conn = None

    def connect(self):
        """Establish SMB connection."""

    def disconnect(self):
        """Close SMB connection."""

    # ... implement WalkerBase methods
```

## Appendix A: smbprotocol Usage Example

```python
from smbprotocol.connection import Connection
from smbprotocol.session import Session
from smbprotocol.tree import TreeConnect

# Connect
conn = Connection(uuid.uuid4(), "192.168.1.10", 445)
conn.connect()

# Authenticate
session = Session(conn, "username", "password")
session.connect()

# Connect to share
tree = TreeConnect(session, "share_name")
tree.connect()

# List directory
```
